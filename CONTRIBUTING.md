# Como Contribuir para o Clean Flow Mentor

Primeiramente, muito obrigado por dedicar seu tempo para contribuir e melhorar essa ferramenta! 🎉 Este é um projeto _Open Source_ e toda ajuda (seja reportando bugs, melhorando a documentação, ou escrevendo código) é muito bem-vinda.

## 🐛 Reportando Bugs

Se você encontrou um erro, por favor abra uma **Issue** no nosso repositório no GitHub. Ao abrir a issue, inclua:

- Uma descrição clara do problema.
- Passos para reproduzir o erro.
- O comportamento esperado vs. o comportamento atual.
- Versão do VS Code e da extensão.

## 💡 Sugerindo Melhorias

Tem uma ideia para tornar o _mentor_ ainda melhor no Clean Code (para PHP ou JS)? Abra uma **Issue** com a tag `enhancement` e descreva como a sua sugestão vai agregar valor aos desenvolvedores.

## 🛠️ Contribuindo com Código

Caso decida colocar a mão na massa, siga este roteiro básico:

1. Faça um **Fork** do repositório.
2. Clone o seu fork localmente: `git clone https://github.com/MariaEduardaLeal/clean-flow-mentor`
3. Instale as dependências executando: `npm install`
4. Crie uma nova branch para a sua feature/correção: `git checkout -b minha-nova-feature`
5. Faça as alterações desejadas. Lembre-se de manter o padrão de código já existente!
6. Teste localmente (pressionando `F5` no VS Code ou rodando a rotina de testes, se houver).
7. Faça o _commit_ das suas alterações (buscamos usar _Conventional Commits_, ex: `feat: adiciona nova regra de clean code`).
8. Suba as alterações para o seu fork: `git push origin minha-nova-feature`
9. Abra um **Pull Request (PR)** para a branch `main` do repositório original. Descreva de forma clara e objetiva o que o seu PR faz.

Mais uma vez, obrigado pelo interesse em fazer este projeto crescer! 🚀
